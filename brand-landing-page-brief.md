# SONICITE 新落地页文档 v0

## 1. 我的理解

这次要做的不是一个单产品首页，而是一个**母品牌落地页**。

参考 [bpmmusic.io](https://bpmmusic.io/) 的原因，不是因为它是 DJ 风格，而是因为它做对了三件事：

- 先讲清楚母品牌是什么
- 再把不同产品线清晰分流
- 用强视觉和极简结构，快速建立“这是一个音乐品牌体系”的认知

结合我查看到的资料，我对你们当前品牌的理解是：

- `Sonicite` 更偏向 DJ / 音乐创作者 / 音乐判断与编排工作流
- `Atmos` 更偏向商业空间 / 品牌方 / 门店场景的声音氛围系统
- 两个产品表面上服务不同人群，但底层共用的是同一个能力方向：
  把原本依赖经验和直觉的“声音判断”，变成一个可理解、可组织、可复用的系统

所以这个新落地页的任务不是“介绍两个产品”，而是完成下面这条叙事：

**SONICITE 是一个做声音智能系统的品牌。**
**Sonicite 面向音乐工作流。**
**Atmos 面向商业空间氛围。**

如果这个定位成立，首页应该先建立品牌认知，再引导用户选择进入 `Sonicite` 或 `Atmos`。

---

## 2. 品牌架构建议

这是这次首页最关键的一点。

目前从现有材料看，更合理的讲法不是把 `sonicite` 和 `atmos` 当成完全平级的两个品牌，而是：

- 母品牌：`SONICITE`
- 产品线 1：`Sonicite` 或 `Sonicite Flow`
- 产品线 2：`Atmos`

原因：

- `Sonicite` 本身已经承载了品牌含义
- `Atmos` 在产品感上更像 SONICITE 下面的一条垂直产品线
- 这样首页叙事更稳，不会让用户困惑“到底谁是公司，谁是产品”

如果你们内部确认就是两个平级产品，也能做，但首页文案必须额外解释：

- `Sonicite` for DJs / music workflows
- `Atmos` for brands / spaces

我建议优先采用前一种。

---

## 3. 首页的核心目标

新落地页需要完成 5 个目标：

- 让用户在 5 秒内知道 SONICITE 是做什么的
- 让不同人群快速找到自己的入口
- 让 `Sonicite` 和 `Atmos` 的区别足够清楚
- 让两个产品又能被看成同一个品牌能力的延伸
- 把流量导向试用、预约 Demo、联系或具体产品页

一句话说，这个首页不是“功能介绍页”，而是一个**品牌入口页 + 产品分流页**。

## 4. 目标用户

### Sonicite 方向

- DJ
- 音乐策展人
- 表演型创作者
- 需要做选歌、理解音乐、排歌、控能量的人

### Atmos 方向

- 咖啡店、餐厅、零售、酒店、健身等空间经营者
- 品牌主理人
- 想把“声音”从随便放歌升级成品牌氛围系统的人

### 首页面对的不是深度用户，而是第一次认识你们的人

所以首页要先回答：

- 你们是谁
- 你们到底解决什么问题
- 为什么会有两个产品
- 我应该点哪一个

## 5. 页面定位

### 首页的一句话定位

**把声音从素材，变成系统。**

或者更完整一点：

**SONICITE 用 AI 理解音乐、场景与判断，把声音变成可设计、可编排、可运营的系统。**

这个定位能同时覆盖：

- Sonicite 的“音乐理解与编排判断”
- Atmos 的“品牌声音与空间氛围系统”

## 7. 页面叙事逻辑

推荐采用这条叙事主线：

### 第一层

声音不是播放列表，而是一个系统。

### 第二层

这个系统在不同场景里，有两种落地方式：

- 面向 DJ 和音乐工作流的 `Sonicite`
- 面向商业空间和品牌氛围的 `Atmos`

### 第三层

两个产品不是随便并列的，它们共享同一种核心能力：

- 理解声音
- 结构化判断
- 根据场景进行编排

这样首页看起来就不是“两块业务拼一起”，而是“一个品牌能力往两个方向展开”。

## 8. 各区块详细文档

## 8.1 Hero 首屏

### 目标

- 先建立 SONICITE 是什么
- 暗示这是音乐相关，但不只属于 DJ 圈
- 立刻抛出两个入口

### 内容结构

- 左上：SONICITE logo
- 右上：简洁导航
- 背景：动态影像或情绪化视频
- 主文案：品牌总标题
- 副文案：一句话解释品牌
- CTA：进入 Sonicite / 进入 Atmos / 预约 Demo

### 建议文案 v1

Eyebrow:

`AI Sound Systems`

主标题：

`让声音从素材，变成系统。`

副标题：

`从 DJ 的选歌与编排，到商业空间的品牌氛围，SONICITE 用 AI 理解音乐、场景与判断。`

按钮：

- `Explore Sonicite`
- `Explore Atmos`
- `Book a Demo`

### 设计建议

- 借鉴 `bpmmusic.io` 的强氛围首屏，但不要只用俱乐部素材
- 建议采用“双世界交替”的视频语言
  一组是设备、波形、DJ、曲库
  一组是咖啡店、餐厅、门店光线、空间氛围
- 整体调性应更像“智能声音品牌”，而不是“夜店内容平台”

## 8.2 Brand Thesis 品牌总论

### 目标

- 解释为什么一个品牌下面会有两个产品
- 建立统一能力认知

### 建议标题

`One brand. Two sound systems.`

或

`同一个声音智能内核，两种产品落地。`

### 建议正文

`我们不是在做“更多音乐功能”。`
`我们在做的是，把原本依赖经验和直觉的声音判断，变成可理解、可编排、可复用的系统。`

`对于 DJ 和音乐工作流，这意味着更快理解音乐、做出更稳定的选歌与编排判断。`
`对于商业空间，这意味着把声音从随机播放，升级为真正服务品牌和营业节奏的氛围系统。`

### 设计建议

- 采用大字号编辑感排版
- 左边一句核心宣言，右边一段解释
- 可以在背景中放很轻的 waveform / grid / light beam

## 8.3 Product Split 双产品入口

### 目标

- 这是首页最重要的分流区
- 用户在这里明确知道自己该点哪里

### 结构

左右两个大卡片：

#### 卡片 A: Sonicite

Label:

`For DJs & Music Workflows`

标题：

`Sonicite`

一句话：

`更快理解音乐，更稳定做选歌与编排判断。`

三点说明：

- 深度解析 BPM、调性、能量、情绪与结构

- 基于场景的 set 编排与判断辅助

CTA:

`View Sonicite`

#### 卡片 B: Atmos

Label:

`For Brands & Spaces`

标题：

`Atmos`

三点说明：

CTA:

`View Atmos`

- hover 时分别进入对应世界观

## 8.4 Sonicite 深入区

### 目标

- 让 DJ / 创作者用户看完后觉得“这是给我用的”
- 讲的是 workflow，不是泛 AI

### 推荐标题

`为音乐判断而设计。`

副标题：

`Sonicite 帮你理解曲库、找到对的歌、组织对的 flow。`

### 核心内容建议

模块 1：Track Intelligence

- 自动识别 BPM、Key、Energy、Mood、Structure
- 把曲库从“凭记忆”变成“可搜索、可理解”

模块 2：Natural Language Search

- 支持用自然语言描述想要的感觉
- 例如“适合日落时段，暖但不散的歌”

模块 3：Set Builder

- 按场景组织曲目
- 帮助处理过渡、能量推进、风格兼容

模块 4：AI Guide

- 给出建议，而不是替代判断
- 强调“辅助创作”，不是“一键生成”

### 建议文案 v1

标题：

`理解音乐，不再只靠反复试听。`

正文：

`Sonicite 把 DJ 长期依赖经验与记忆完成的工作，变成一个更清晰的工作流。`
`从曲库理解、搜歌、排歌到混音判断，你可以更快进入状态，把时间留给真正重要的表达。`

### 建议展示素材

- 曲库分析界面
- 搜歌界面
- set builder 界面
- 音乐标签或能量曲线可视化

---

## 8.5 Atmos 深入区

### 目标

- 让品牌方和空间经营者看懂产品价值
- 强调这不是“给店里放歌”，而是“做声音系统”

### 推荐标题

`为每个空间，设计自己的声音秩序。`

副标题：

`Atmos 根据品牌、空间类型、客群与营业节奏，生成可持续运行的声音蓝图。`

### 核心内容建议

模块 1：Brand Onboarding

- 输入品牌调性、客群、空间信息、参考音乐
- 不要求用户懂音乐技术语言

模块 2：Sound Blueprint

- 按营业时段生成全天的氛围策略
- 告诉用户何时更克制、何时更提神、何时更适合停留

模块 3：Atmosphere Stream

- 不是一首首歌的列表
- 而是无感流动的长时段氛围流

模块 4：Player / Control

- 用户通过“更平静 / 更活跃”这类简单控件调节空间状态

### 建议文案 v1

标题：

`不是歌单，而是品牌氛围系统。`

正文：

`Atmos 把声音从“随便放点音乐”升级成可被设计和管理的品牌资产。`
`它理解你的空间、客群和营业节奏，为不同时段生成匹配的氛围流，让声音真正服务生意和体验。`

### 建议展示素材

- onboarding 问卷界面
- sound blueprint 曲线
- 分时段卡片
- player 界面

---

## 8.6 Shared Engine 共用能力区

### 目标

- 把两个产品重新收回到一个品牌逻辑里
- 建立技术与方法论上的统一性

### 推荐标题

`The same intelligence, applied differently.`

或

`同一个声音智能引擎，服务不同场景。`

### 建议结构

三步流程：

1. Understand
2. Structure
3. Orchestrate

对应解释：

1. Understand
   识别音乐、品牌、场景与上下文

2. Structure
   把感性的判断转成清晰的标签、蓝图和规则

3. Orchestrate
   生成适合场景的推荐、编排、氛围流或播放体验

### 这个区块很重要

如果没有这一段，首页会像两个独立产品拼接。
有了这一段，首页才会更像一个真正的品牌官网。

## 8.7 Demo / Use Cases 场景展示区

### 目标

- 不靠空话，给用户一个“我知道它怎么用”的直觉

### 推荐做法

用 4 个真实感较强的场景卡片，而不是生硬的功能列表。

建议场景：

- `Build a better set for a late-night club`
- `Find the right tracks for a sunset opening`
- `Design a calmer afternoon for a cafe`
- `Shift a restaurant from lunch focus to evening warmth`

### 设计建议

- 每张卡片一张静态图或短视频
- 一句场景描述
- 一个输出结果

例如：

`Input: 品牌调性 + 客群 + 参考音乐`
`Output: Afternoon blueprint + playable stream`

## 9. 视觉方向建议

## 9.1 可以借鉴 bpmmusic.io 的部分

- 全屏沉浸式首屏
- 极简导航
- 强品牌感的黑底视频视觉
- 首页作为母品牌入口，而不是单产品销售页
- 清晰的人群 / 产品分流

## 9.2 不建议照搬的部分

- 过强的俱乐部或 DJ 单一氛围
- 明显偏“record pool / content subscription”的语气
- 纯销售式的“登录 / 注册”导向

你们更适合的气质是：

- 更克制
- 更智能
- 更偏编辑感
- 更强调“判断”和“系统”
- 同时能容纳音乐创作与商业空间两种语境

## 10. 视觉语言建议

### 关键词

- Cinematic
- Intelligent
- Editorial
- Precise
- Warm-tech

### 色彩建议

- 主底色：Carbon Black
- 主文字：Soft Ivory
- Atmos 辅助色：Warm Amber / Bronze
- Sonicite 辅助色：Electric Blue / Steel Cyan

### 字体建议

- 标题：高对比衬线或带编辑感的 serif
- 正文与 UI：现代 grotesk sans

推荐方向：

- `Cormorant Garamond` + `Neue Montreal`
- 或 `Canela` 风格 + `Satoshi`

### 动效建议

- 首屏视频 crossfade
- 滚动时产品卡片轻微视差
- 声波 / 光束 / 粒子作为弱辅助，而不是主视觉噱头
- 两个产品区块切换时，冷暖调性明显变化
